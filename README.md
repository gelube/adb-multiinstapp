# ADB 批量管理工具 v2.0

用于在局域网内批量发现 ADB 设备并进行应用安装/卸载管理的 Windows 桌面工具。

## ✨ v2.0 新功能

| 功能 | 说明 |
|---|---|
| 🔧 **自定义端口** | 支持修改 ADB 端口，不再固定 5555 |
| 📊 **版本检测** | 自动对比已安装版本和新版本，智能决定是否安装 |
| 🔄 **失败重试** | 安装失败的设备可单独选择重试，自动清理后重装 |
| 📋 **详细日志** | 增强日志显示，支持导出 |

## 🎯 功能特性

### 📱 设备发现
- ✅ IP 范围扫描（如 192.168.1.100 - 192.168.1.200）
- ✅ **自定义 ADB 端口**（默认 5555，可修改）
- ✅ 可配置最大并发扫描数（默认 50 线程）
- ✅ 实时显示设备信息（IP、状态、型号、Android 版本）

### 📦 应用安装
- ✅ 选择 APK 文件进行批量安装
- ✅ **APK 版本信息自动读取**（versionCode/versionName）
  - 使用纯 Python 解析，无需额外工具
  - 支持标准格式的 APK 文件
  - 部分加固/混淆的 APK 可能无法自动识别版本
- ✅ **已安装版本检测**，显示在设备列表
- ✅ **智能版本策略**：
  - **智能对比**：自动对比版本，只有新版本才安装
  - **跳过已安装**：只要已安装就跳过
  - **强制覆盖**：始终覆盖安装
  - **先卸载再安装**：彻底重装
- ✅ 可配置同时安装的设备数量（默认 10 台）
- ✅ 实时进度显示和结果统计

### ⚠️ 失败重试（新增）
- ✅ **自动记录安装失败的设备**
- ✅ **显示失败原因**
- ✅ **可选择部分或全部重试**
- ✅ **重试前自动卸载旧版本**
- ✅ **记录重试次数**

### 🗑️ 应用卸载
- ✅ 输入包名批量卸载
- ✅ 自动跳过未安装设备
- ✅ 并发控制，实时进度显示

### 📋 日志管理
- ✅ 实时日志窗口（深色主题）
- ✅ 支持导出日志文件

## 📦 安装步骤

### 1. 安装 Python
确保已安装 Python 3.8 或更高版本。

### 2. 安装 ADB
下载 Android SDK Platform-Tools：
- 官网：https://developer.android.com/studio/releases/platform-tools
- 解压后将 `adb.exe` 所在目录添加到系统 PATH

验证 ADB 安装：
```bash
adb version
```

### 3. 安装依赖
```bash
cd adb-batch-manager
pip install -r requirements.txt
```

依赖说明：
- **PyQt5**: GUI 界面
- **pure-python-adb**: ADB 连接（可选）
- **apkutils2**: APK 文件解析（用于读取版本信息）

> 💡 **提示**：如果 `pip install` 失败，可以尝试使用 `uv pip install -r requirements.txt`

### 4. 运行程序
```bash
# 方式 1: 双击 run.bat
# 方式 2: 命令行
python adb_manager.py
```

## 📖 使用说明

### 设备发现
1. 输入 IP 范围（如 100-200，表示 192.168.1.100-200）
2. **修改 ADB 端口**（如设备使用 5556）
3. 设置并发数（建议 50）
4. 点击"开始扫描"

### 应用安装
1. 切换到"应用安装"标签
2. 选择 APK 文件（自动读取版本信息）
3. 输入应用包名
4. **选择版本策略**：
   - 智能对比：推荐，自动判断是否需要安装
   - 跳过已安装：不检查版本
   - 强制覆盖：始终安装
   - 先卸载再安装：彻底重装
5. 选择要安装的设备
6. 点击"开始安装"
7. **失败的设备会自动出现在"失败重试"标签**

### 失败重试
1. 安装完成后，失败设备自动出现在此页面
2. 查看失败原因
3. 选择要重试的设备
4. 点击"重试选中设备"
5. 重试前会自动卸载旧版本

### 应用卸载
1. 切换到"应用卸载"标签
2. 输入要卸载的包名
3. 选择要卸载的设备
4. 点击"开始卸载"

## ⚙️ 配置说明

| 设置 | 默认值 | 说明 |
|---|---|---|
| 扫描并发数 | 50 | 同时扫描的 IP 数量 |
| ADB 端口 | 5555 | **可自定义修改** |
| 安装并发数 | 10 | 同时安装的设备数 |
| 版本策略 | 智能对比 | compare/skip/force/reinstall |

## 🔧 技术栈

- **语言**: Python 3.8+
- **GUI 框架**: PyQt5
- **ADB 交互**: subprocess 调用系统 ADB
- **并发处理**: concurrent.futures.ThreadPoolExecutor
- **版本检测**: aapt 解析 APK + dumpsys 获取已安装版本

## ⚠️ 注意事项

1. **设备准备**：确保目标设备已开启 USB 调试和无线调试
2. **网络连接**：确保电脑和设备在同一局域网
3. **ADB 授权**：首次连接设备需要授权
4. **防火墙**：确保防火墙未阻止 ADB 连接
5. **aapt 工具**：版本检测需要 aapt（Android SDK Build-Tools）
